Implementation of two dimensional lists of primitives doesn't really differ from one-dimensional lists in recipe config, but it is somewhat expected that your nested data has the same length.

```json title="colors.recipe.json" 
{
  "name": "colors",
  "type": "CORE:UiRecipe",
  "description": "List of colors",
  "plugin": "@development-framework/dm-core-plugins/data_grid",
  "config": {
    "type": "PLUGINS:dm-core-plugins/data_grid/DataGridPluginConfig",
    "fieldNames": [
      "colors"
    ],
    "columnLabels": [
      "NAME",
      "HEX",
      "RGB"
    ],
    "showRows": false
  }
} 
```
```json title="ColorList.blueprint.json" 
{
  "name": "ColorList",
  "type": "dmss://system/SIMOS/Blueprint",
  "attributes": [
    {
      "name": "type",
      "type": "dmss://system/SIMOS/BlueprintAttribute",
      "attributeType": "string"
    },
    {
      "name": "colors",
      "type": "dmss://system/SIMOS/BlueprintAttribute",
      "attributeType": "string",
      "dimensions": "*,*"
    }
  ]
} 
```

```json title="colors.entity.json" 
{
  "type": "./ColorList",
  "colors": [
    [
      "red",
      "#FF0000",
      "rgb(255, 0, 0)"
    ],
    [
      "blue",

      "rgb(0, 0, 255)"

    [
      "green",
      "#00FF00",
      "rgb(0, 255, 0)"
    ]
  ]
} 
```
